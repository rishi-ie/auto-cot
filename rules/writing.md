# Rule: Writing

## When to use
Task involves: writing prose, documentation, emails, reports, blog posts, technical writing, user guides, or any task where the output is primarily text meant for humans to read.

## Trigger keywords
write, document, explain, describe, summarize, outline, draft, edit, revise, create content, compose, author, craft

## Reasoning steps

### 1. Define the audience and purpose
Before writing anything:
- Who will read this?
- What do they already know?
- What do they need from this?
- What should they do or decide after reading?
- How will this be consumed? (silent reading, presentation, voice)

### 2. Choose structure first
Pick the right structure for the content type:
- **Chronological**: events over time, tutorials, histories
- **Problem-solution**: issue, impact, resolution, benefits
- **Comparative**: A vs B, pros and cons, tradeoffs
- **Hierarchical**: big picture → details → summary
- **Inverted pyramid**: conclusion first, details second, context third

### 3. State the thesis
One clear statement of what this piece is about.
- "This document explains X to Y so they can Z"
- If you can't state the thesis in one sentence, you don't know what you're writing

### 4. Write the skeleton
Outline before drafting:
- Introduction (hook + thesis + preview)
- Body sections (each with a clear point)
- Conclusion (summary + action/decision if needed)

### 5. Draft with the thesis
Write fast, edit later. Follow your outline.
- Keep the thesis in mind for every paragraph
- If a paragraph doesn't serve the thesis, cut it
- Write transitions between sections

### 6. Edit ruthlessly
After the draft exists:
- Remove every word that doesn't add meaning
- Each sentence should be the simplest it can be
- Each paragraph should be the shortest it can be
- Read it aloud — if you stumble, rewrite

### 7. Verify
Before finishing:
- Does it achieve the stated purpose?
- Is the audience addressed directly?
- Is the structure appropriate for the content?
- Are there any ambiguous terms left unexplained?
- Does it flow from section to section?

## Red flags
- Writing before knowing who will read it
- No clear structure — just paragraphs of text
- Vague sentences that could mean multiple things
- Self-indulgent prose that doesn't serve the reader
- Forgetting the call to action or decision point

## Exit criteria
- [ ] Audience and purpose stated at the top
- [ ] Structure appropriate for content type
- [ ] Every section serves the thesis
- [ ] Prose is clear, direct, and as short as possible
- [ ] Readable aloud without stumbling
- [ ] Ready for review by the intended audience
